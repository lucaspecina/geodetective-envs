- Evaluación de viabilidad técnica con Codex en `research/synthesis/viability_assessment.md`. Hallazgos críticos: (a) Google Maps TOS prohíbe explícitamente usar Maps Content para training/validating ML — fuerza re-arquitectura "stack primario open, Google opcional con user key + non-training default"; (b) reverse image search web-scale es el bloqueador más serio — ningún proveedor cubre Lens/Yandex a costo razonable, requiere filtrado tiered best-effort; (c) OpenHistoricalMap promovido a v1 (no v1.5).
- **Pivote a framing benchmark**: foco actual del proyecto = **benchmark de evaluación**, no env de training. La idea env queda como deuda futura. Disclaimer agregado al inicio de `PROJECT.md`. Invariante 7 (TOS) reformulada para distinguir uso benchmark (defendible) vs training (prohibido). Notas agregadas en `research/synthesis/{viability_assessment,related_work_decisions}.md` indicando el shift. Naming "GeoDetective Envs" → eventual "GeoDetective Benchmark" queda como deuda explícita.
- **Primer código operacional** (`src/geodetective/` + `scripts/`): conda env `geodetective` con Python 3.11, integración con Azure Foundry (gpt-4o/4.1/5/5.4), Tavily para web search, PIL/geopy/httpx. Todo bajo el .env compartido con arcagi3 (mismas credenciales).
- **Experimento E001** (`research/notes/E001_test3_no_tools_results.md`): Test 3 (VLM sin tools, N=3 runs) sobre 17 fotos PastVu. Resultado: filtro v2 simplificado (source + dist_min<10km AND conf≥media) deja sobrevivir ~53%. Sweet spot confirmado: fotos cotidianas sin landmark + rurales URSS/Cáucaso/Kazajstán + pre-1900.
- **Experimento E002** (`research/notes/E002_react_websearch_first_run.md`): primer ReAct + web_search filtrada sobre #1748874 (SP barrio anónimo). Resultado: **2573 km sin tools → 8.5 km con web_search ReAct (mejora 300x)**. Concepto del benchmark VALIDADO end-to-end.
- **Plan de validación revisado** (`research/synthesis/validation_plan.md`): 6 fases con gates falsables, post-crítica Codex (sin fase legal — postergada). Estimado: 12-15 semanas hasta v1 publicable.
- **Stack completo de 11 tools del agente** (`src/geodetective/tools/`): web_search (advanced), fetch_url, fetch_url_with_images (con imagehash), image_search (con hash perceptual flag), geocode + reverse_geocode (Nominatim), historical_query (OpenHistoricalMap Overpass — pieza diferencial), crop_image + crop_image_relative (PIL local), static_map (Google Static Maps), street_view (Google Street View Static).
- **ReAct loop expandido** (`src/geodetective/agents/react.py`): inyección de imágenes en messages cuando una tool devuelve visuales. Hash perceptual `phash` de la foto target con flag `is_likely_target` (no bloqueo, solo flag). Counters por tool en ReActResult. Max steps subido a 12.
- **Google Maps Platform integrado**: Maps Static + Street View Static APIs. Free tier $200/mes. Cuotas restrictivas configuradas (500/día Static, 200/día Street View). Cobertura SV confirmada en BA, Volga rural, SP barrio.
- **Experimento E003** (`research/notes/E003_react_full_tools.md`): batch parcial 9/15 runs (cancelado por costo Tavily) + test 1 run con stack completo. Hallazgos: (a) tools NO siempre ayudan — eso ES señal del benchmark; (b) modelo a veces ignora tools óptimas (no usó street_view en el caso Strahov→1493 km); (c) decisión: NO forzar uso de tools en system prompt (sesgaría benchmark).
