- **#24 cerrado — atacante adversarial GPT-4o + deuda hash perceptual**. Cierra el epic #21 (pipeline de filtrado). Nuevo script `scripts/run_attacker_filter.py`: lee las 180 fotos de E007, baja+limpia cada una (vía `clean_image` de #22), corre N=3 llamadas a `gpt-4o` sin tools en paralelo (ThreadPoolExecutor, 8 workers), parsea lat/lon/confidence, computa distancia geodésica vs ground truth, y descarta si en ALGUNA corrida cumple `dist<10km AND conf in {media, alta}`. Cada result registra `triggering_runs` con índices de las corridas que dispararon (Codex review). Resultado: **101 keep (56.1%), 79 reject (43.9%), 0 skip**. 60/79 rejects (76%) dispararon las 3 corridas — patrón consistente, no ruido. Tiempo: 386s (6.4 min) con 8 workers. Distribución por bucket país: Norteamerica reject_rate 80% (most familiar para gpt-4o), Russia-Asia 13% (menos), Russia-EU 27%, Ex-URSS 37%, Europa-no-URSS 50%, Resto 57%. Por década: 1890s/1900s 53-57% (landmarks), 1930s 30% (más cotidianas). **Deuda hash perceptual implementada** en `src/geodetective/agents/react.py`: cuando `image_search` o `fetch_url_with_images` devuelven imagen con `is_likely_target=True`, ahora se OCULTA del modelo (hard reject): ni bytes ni URL — solo se informa cuántas se ocultaron. Alinea código con `PROJECT.md`. Iteración Codex (1 ronda): aplicados 3 puntos — (a) redacción de URL en `images_metadata` para target matches (no solo bytes; el dominio del match es shortcut), (b) `triggering_runs` para mejor signal en analysis, (c) lenguaje del audit moderado (Norteamerica 80% es "consistent with" familiaridad GPT-4o + landmark-prevalence, no causal sin control). Output completo: `research/notes/E004_attacker_filter.md`. Próximo: ya no hay sub-issues abiertas del epic #21, queda como issue futura escalar K_PER_CELL para corpus final balanceado.
