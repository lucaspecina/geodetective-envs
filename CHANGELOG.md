# Changelog

## 2026-05-11

- **#17 cerrado — sample diverso del corpus PastVu**. Nuevo script `scripts/sample_diverso.py`: lee el dump bajado en #3, filtra eligibles (type=1 + geo + year + 1890-1949) → 596K candidatos, asigna cada uno a celda `(bucket_pais, bucket_decada)` (6×6 = 36 celdas), de-duplica por geohash5 (~5km) dentro de cada celda y sortea K=5 fotos por celda con seed=42. Output: `experiments/E007_sample_diverso/candidates.json` (180 fotos, todas las celdas llenas) + `audit_summary.json`. Buckets país: `Russia-EU` (lon<60°), `Russia-Asia` (lon≥60°), `Ex-URSS` (14 países post-soviéticos), `Europa-no-URSS` (33 países incl. microestados), `Norteamerica` (USA/Canada), `Resto`. Décadas: 1890s-1940s. Iteración Codex (1 ronda review): aplicados 4 puntos — microestados europeos agregados a EUROPA_NO_URSS (Mónaco antes caía en Resto), schema notes para consumidores futuros (`file_url` no `file`, `provenance_source` no `source`), dedup intra-cell documentada (mismo geohash puede aparecer en distintas décadas, intencional), confirmación de determinismo del shuffle+seed. Doc en `research/notes/sample_diverso_audit.md`. Notable: sample incluye Wall Street 1895, Casino de Monte-Carlo 1894, Nanking Road Shanghai 1890 — landmarks famosos que el atacante de #24 debería detectar como trampeables.
- **#3 cerrado — audit de metadata PastVu**. Bajado `pastvu.jsonl.zst` (282 MB, HF `nyuuzyou/pastvu`) y streameado completo en 35s (`scripts/audit_pastvu_metadata.py`). Findings clave: 2.08M records (98% fotos type=1), 97% con `geo`, 99.95% con `year`, 100% con país (regions[0]). Sesgo Rusia: 62% (74% ex-URSS) — menos extremo que genesis-intro afirmaba (70-95%). **100% de las fotos tienen watermark** (`waterh > 0`) — `clean_image` (#22) confirmado como obligatorio. **676K fotos elegibles** (type=1 + geo + year + 1850-1950). 2094 shards de imágenes confirmados (no 47 como decía pastvu_deep_dive.md — esa snapshot estaba vieja). Doc completo en `research/notes/pastvu_metadata_audit.md`. Iteración Codex (1 ronda): refinamientos aplicados — 6 buckets de país (no 5, USA con masa propia), audit de campos extra (`regions` depth, `s`, `dir`, `year2`, dimensiones). **Cambio de decisión post-audit (user)**: urbano/rural soltado como dimensión de balance — no hay field directo y la derivación es frágil, país × década alcanza para v1. Updates a bodies de #21 y #17 reflejando esto.
- **Decisiones operativas del pipeline de filtrado (#21)** cerradas y documentadas en bodies de #21 (epic), #24 (atacante) y #17 (sample diverso): (a) "trampeable" = A (pistas escondidas) + B (visualmente obvia) + C (soluble por memoria), con B≈C en práctica (B se detecta porque cae en C); (b) modelo atacante = `gpt-4o` sin tools, 1 modelo en v1 (no multi-modelo, deuda documentada); (c) umbral = `dist_min<10km AND conf≥media` en N=3 corridas (mismo criterio que filtro v2 E001); (d) balance del sample = país/región × década × urbano/rural + de-clustering geográfico (geohash/H3) permanente; (e) hash perceptual match con target = hard reject (no flag) — alinear `react.py` con `PROJECT.md` queda como deuda explícita en #24. Mini threat model (5 filas: EXIF / watermark / source domain / reverse-search / modelo-de-memoria) agregado al body de #21 — cubre lo necesario para arrancar, #10 completo sigue abierto pero no bloquea. Orden de ejecución del pipeline: #3 → #17 → #24.

## 2026-05-08

- **Blacklist runtime per-photo** (#23): nuevo módulo `src/geodetective/corpus/blacklist.py` con `BLOCKED_DOMAINS_GLOBAL` (frozenset minimal: reverse search engines + agregadores con metadata estructurada + hosting/sharing) + `PROVIDER_DOMAINS` dict (pastvu, smapshot, etoretro, humus, oldnyc, oldsf, historypin, sepiatown) + `compute_excluded_domains(provider, source, extra)` + `is_blocked(url, excluded_domains)` con `urlparse` + suffix match (sin substring false positives). Las 4 tools del agente (`web_search`, `fetch_url`, `fetch_url_with_images`, `image_search`) aceptan `excluded_domains` y rechequean URL post-redirect. `web_search` overfetcha *3 a Tavily. ReAct loop computa `excluded_domains` una vez desde provider+source y lo pasa a cada tool call. SYSTEM_PROMPT actualizado para no enumerar dominios bloqueados. `sample_pastvu.py` agrega field `provider="pastvu"`. Tests sintéticos en `scripts/test_blacklist.py`: 14 grupos, 65 checks, cubren suffix match, GLOBAL, per-provider, source schemeless, E2E, anti-overblocking. Iteración Codex (2 rondas review): 8 bugs reales aplicados (substring → suffix, post-redirect, SYSTEM_PROMPT mintiendo, pre-filter `max_results`, `<img>` con final_url, google/bing/googleusercontent overblocking, schemeless URLs, back-compat semántica). **Cambio importante**: `BLOCKED_DOMAINS` de `web_search.py` ya NO contiene pastvu/smapshot/etc — pasa per-photo. Alias retenido para back-compat de imports.
- **Paso 0 del filtrado implementado** (#22, cierra #18): nuevo módulo `src/geodetective/corpus/clean_image.py` con `clean_image(raw_path, provider, provider_meta) -> CleanResult`. Hace strip EXIF + crop watermark por regla de provider + RGBA→RGB sobre fondo blanco + re-encode JPEG strippeando ICC/comments/XMP. Escritura atómica (temp + replace). Filename versionado `_clean_v{N}.jpg` para invalidar cache cuando cambia la lógica. Refactor de `scripts/test3_no_tools.py:download_photo()` y `scripts/run_react_websearch.py` para consumirlo. Tests sintéticos en `scripts/test_clean_image.py`: 13 escenarios (T1-T13) cubriendo EXIF, RGBA, P+transparency, ICC, JPEG comment, metadata patológica (waterh≥h), ceil clamp, cache. Iteración con Codex (1 ronda review): aplicados 5 issues reales — `int()` floor → `math.ceil()` + clamp, metadata patológica → `discarded` en vez de pass-through, modos `PA/La/RGBa` al composite blanco, write atómico con `os.replace()`, `img.info` se carrea en JPEG save → re-crear imagen desde pixels.
- **Diseño operativo del pipeline de filtrado del corpus** documentado como issue #21 (epic) + 3 sub-issues accionables. Decisiones canónicas: (a) atacante adversarial barato con modelo chico en 3 rondas (sin tools / con búsqueda / TinEye sampleado), no solo Test 3 estático; (b) Paso 0 de limpieza pre-filtro con strip EXIF + crop watermark por regla de fuente + RGBA→RGB (cierra #18, sub-issue #22); (c) blacklist de dominios runtime cambia de global a per-photo — el resto del archivo histórico queda accesible para investigación legítima (sub-issue #23); (d) Paso 2A multi-modelo en sample diverso antes de aplicar el filtro a escala, evita overfit del corpus a un solo modelo (sub-issue #24).
